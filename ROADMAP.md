## 📋 Issues驱动开发流程

### 1. Issue创建规范
- 使用标准模板 (Bug/Feature/Task)
- 明确标签分类 (priority/size/type/area)
- 详细的验收标准
- 合理的时间估算

### 2. 开发流程
```
Issue创建 → 需求澄清 → 技术设计 → 开发实现 → 代码审查 → 测试验证 → 部署发布 → Issue关闭
```

### 3. 项目看板状态
- **📋 Backlog**: 待规划的Issues
- **🔍 In Analysis**: 需求分析中
- **⚡ Ready**: 准备开发
- **🔨 In Progress**: 开发中
- **👀 In Review**: 代码审查中
- **🧪 Testing**: 测试验证中
- **✅ Done**: 已完成

### 4. 标签体系
- **优先级**: `priority/high` `priority/medium` `priority/low`
- **规模**: `size/small` `size/medium` `size/large`
- **类型**: `type/feature` `type/bugfix` `type/refactor` `type/docs`
- **领域**: `area/core` `area/integration` `area/ui` `area/docs`
- **状态**: `status/blocked` `status/help-wanted` `status/good-first-issue`
