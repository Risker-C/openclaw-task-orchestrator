# 十项目技巧映射（可执行版）

> 来源：上午10个参考项目；目标：沉淀为 Task Orchestrator 可直接使用的规则。

## 1) OpenSpec（规范驱动）
**技巧**：`propose → apply → archive` 三段式。
**落地规则**：
- L3任务先出“任务提案+验收标准”，再执行。
- 完成后归档到飞书文档/Wiki，保留变更轨迹。

## 2) edict（三省六部）
**技巧**：异步工单 + 可观测性。
**落地规则**：
- 主Agent只做受理/编排/回报，不做长耗时执行。
- 所有任务必须有状态与进度，支持中途干预。

## 3) Magi
**技巧**：复杂度智能分流 + 异构模型分工。
**落地规则**：
- 必做L1/L2/L3分流，拒绝“一刀切全并发”。
- 根据任务类型选专长Agent，降低无效token消耗。

## 4) CCG-workflow
**技巧**：智能路由 + 中心审核。
**落地规则**：
- 前后端/研究/文档任务走不同Agent路由。
- 关键产出引入主Agent最终审核后再对外。

## 5) GuDaStudio/skills
**技巧**：分阶段协作 + Resource Matrix。
**落地规则**：
- 每种任务有固定阶段模板（分析→执行→审查）。
- 明确“谁负责什么”，减少Agent角色重叠。

## 6) superspec
**技巧**：零决策计划（实现阶段不再临场决策）。
**落地规则**：
- L3任务执行前必须明确：文件范围、验证命令、验收标准。
- 存在冲突点时先消解再执行。

## 7) GuDaStudio/commands
**技巧**：RPI（Research/Plan/Implementation）与上下文卫生。
**落地规则**：
- 复杂任务按 R→P→I 三步推进。
- 阶段切换时做摘要，避免上下文膨胀。

## 8) UltimateSearchSkill
**技巧**：双引擎检索 + 证据分级。
**落地规则**：
- 关键事实至少双来源交叉验证。
- 结论标注置信度（High/Medium/Low）。

## 9) AutoRedTeam-Orchestrator
**技巧**：事件驱动 + 可验证结果。
**落地规则**：
- 关键任务结果不能只给结论，必须附验证步骤。
- 错误走统一恢复链路（重试→降级→兜底）。

## 10) Claude-Code-Workflow
**技巧**：Beat事件循环 + 并行编排。
**落地规则**：
- 只在“有事件时”推进编排，避免无意义轮询。
- 并行任务上限受系统限制（当前5活跃子会话/session），分批调度。

---

## 汇总成4条总原则

1. **异步优先**：主Agent永不阻塞。
2. **分流优先**：先判复杂度，再选执行模式。
3. **事件优先**：以状态变化驱动流程推进。
4. **证据优先**：关键结论必须可验证、可追溯。
