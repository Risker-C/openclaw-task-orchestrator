## 快速创建Issues的脚本

为了快速创建这些Issues，你可以使用GitHub CLI：

```bash
# 安装GitHub CLI
# 然后使用以下命令创建Issues

gh issue create --title "[TASK] Implement Basic Task Management API" \
  --body-file issue-templates/task-1.md \
  --label "priority/high,size/medium,type/feature,area/core" \
  --milestone "v0.1.0"

# 重复以上命令创建所有18个Issues
```

## 项目看板配置建议

创建Project Board后，建议配置以下自动化规则：

1. **自动移动规则**:
   - 新创建的Issues自动进入 "Backlog"
   - 分配给开发者的Issues自动移动到 "Ready"
   - 创建PR时自动移动到 "In Review"
   - PR合并后自动移动到 "Done"

2. **标签自动化**:
   - 根据里程碑自动添加版本标签
   - 根据文件路径自动添加area标签
   - 根据Issue模板自动添加type标签

3. **通知设置**:
   - Issues状态变更时通知相关人员
   - 里程碑截止日期提醒
   - 长时间无更新的Issues提醒

这样就建立了完整的Issues驱动开发流程！🚀
