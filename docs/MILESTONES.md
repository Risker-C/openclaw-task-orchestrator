# GitHub 里程碑策略（v0.0.1 起）

## 原则

- 只维护**当前正在执行**的里程碑
- 不提前创建 v0.1.0 / v0.2.0 / v0.3.0 等远期里程碑
- 完成当前版本后，再按实际结果创建下一个小版本（如 v0.0.2 或 v0.1.0）

- **v0.0.1 - 初始版本**

## 下一里程碑创建条件

仅当以下条件满足时，才创建下一里程碑：

- v0.0.1 发布完成
- 真实测试结果稳定
- 已整理并确认新增需求清单
- 明确下一版本目标（修复/增强/功能扩展）

> 版本管理采用“小版本迭代”策略：务实、可验证、可回滚。
